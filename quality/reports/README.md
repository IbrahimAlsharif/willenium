# Quality Reports

Use this folder for reporting formats that communicate release confidence, not only execution status.

## A Strong Report Should Answer

- what critical risks were covered
- which meaningful scenarios were exercised
- what important scenarios remain uncovered
- what the main remaining exposure is
- what confidence level the team can claim
- what decision the report is safe to support

## Avoid Reducing Reporting To

- pass count
- fail count
- raw execution output without risk context

The long-term goal is for Willenium reports to help engineering, product, and leadership make safer release decisions.
